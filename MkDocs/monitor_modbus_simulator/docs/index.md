# 欢迎来到 </br>IBM Maximo Monitor Modbus 模拟器实验
本 Maximo 动手实验将介绍如何安装和设置 Modbus 模拟器，</br>
该模拟器将模拟两个 Lenze i550 变频驱动器（VFD）。

它可以用于许多不同的自定义演示，这就是为什么它是一个单独的实验。

创建本实验的理由是：

- 提供一个易于使用的 modbus 模拟器，可用于其他实验或其他自定义演示。
- 提供来自工业设备（变频驱动器）的一些 OT 数据点 - 而无需实际拥有一个。
- 提供一个多平台模拟器，可在 Windows、MacOS 和 Linux 上使用。
- 提供一个具有动态 OT 数据点的模拟器，而不是仅提供固定值的基于 Windows 的模拟器（例如 Unslave）

## 模拟器
modbus 模拟器正在模拟 2 个独立的 Lenze i550 变频驱动器</br>
提供以下数据点：

| 保持寄存器 | 数据点                            | 类型      |
|:----------------:|:--------------------------------------|:----------|
| 2000             | AC 驱动器状态字                  | 静态    |
| 2001             | 频率                             | 动态   |
| 2002             | 错误代码                            | 静态    |
| 2003             | 驱动器状态                          | 静态    |
| 2004             | 电机电压                         | 动态   |
| 2005             | 电机电流                         | 动态   |
| 2006             | 未使用                              | 静态    |
| 2007             | 未使用                              | 静态    |
| 2008             | 未使用                              | 静态    |
| 2009             | 控制卡温度 C            | 动态   |
| 2010             | 直流母线电压                        | 动态   |
| 2011             | 数字输入状态                  | 静态    |
| 2012             | 电机扭矩                          | 随机    |
| 2101             | 频率命令                     | 静态    |

这两个设备使用 Modbus TCP 协议，使用端口 10502 和 20502。</br>
因此，您需要使用运行模拟器的机器的 IP 地址与端口号组合，例如</br>
- 192.168.1.42:10502</br>
- 192.168.1.42:20502

!!! tip
    动态和随机值将每 30 秒更改一次。</br>
    动态值使用 [Perlin 噪声](https://en.wikipedia.org/wiki/Perlin_noise){target=_blank}创建</br>
    Modbus 设备的标准端口号是 502，但我决定使用端口 10502 和 20502，提供两个独立的设备，可在多个平台上工作。
